The Product

What is
Claude Code?

Claude Code is an AI tool you run from the terminal. It can read your project, edit files, run commands, and help complete coding tasks.

It is basically an AI engineer that sits inside your project folders and takes instructions from you.

The Workspace

What is the
Terminal?

The terminal is basically a text-based control panel for your computer. Instead of clicking buttons, you type instructions.

For non-technical folks: it looks like the Matrix, but it's just a place where you talk to your computer with text instead of a mouse.

Developer Environment

Why Use the Terminal?

The terminal is where developers already build, test, and run things. That makes it the best place for Claude Code, because it can work directly with files, commands, and tools in the same environment.

By living in the terminal, Claude doesn't just write code in a browser window for you to copy-paste. It writes the code directly into your actual files.

Before You Start

What You Actually Need

Before running a single command, make sure you have these in place.

An Anthropic account — claude.ai Pro or Max. Claude Code is not available on the free plan.

Node.js and npm installed — the engine Claude Code runs on. Download from nodejs.org if you don't have it.

A project folder — Claude Code works best when it has real files to read. Don't point it at an empty desktop.

Windows users: install WSL (Windows Subsystem for Linux) first. It makes everything smoother.

Quick Start

Setup:
Simple, Not Always Fast

Three commands. But on Windows especially, getting Node.js, npm, and the terminal playing nicely can take a while. Budget 20–30 minutes your first time.

$ npm install -g @anthropic-ai/claude-code
$ cd any-project-folder
$ claude
> Ready to work.

Mac users: usually smooth. Windows users: expect some friction. It's worth it once it's done.

Session Hygiene

Managing Your Context

/clear

Wipes the conversation history without closing Claude. Use it between unrelated tasks — it's the highest-ROI habit you can build.

/compact

Summarises the conversation so far to free up context space. Use this mid-session when you want to keep going without starting over.

Fresh Session

After two failed corrections, just start a new session. Give Claude a one-paragraph recap. A clean start beats a broken conversation every time.

Think of context like RAM. The more you load in, the slower and fuzzier things get. Keep sessions focused and short.

Security

Keeping It Safe

Claude Code is not magic access to your whole computer. It usually works within your permissions and workspace rules, and sandboxing can restrict what it can change outside the current directory.

Don't worry that it's going to delete your operating system. It works on the folder you put it in.

Safety Net

Checkpoints:
Your Undo Button

Before big changes, Claude saves a checkpoint. If something goes wrong, you can rewind instantly.

Think of it like a save point in a video game. No Git expertise required.

Reality Check

The Honest Truth

CLAUDE.md hints fade over time. Long sessions = fading memory. Keep your brain file to ~1 page.

Use /clear between unrelated tasks. It's the single highest-ROI habit.

After two failed corrections, start fresh. A clean prompt beats arguing in circles.

Always verify the work. Claude is enthusiastic, fast, and occasionally wrong.

Tokens burn fast. Intensive work like coding can drain your token limit quickly, especially on the Pro plan. Always plan your task before you start, not halfway through.

Troubleshooting

When Things Go Wrong

Claude is going in circles

Stop it. Type /clear and restate the goal more simply. Arguing with a looping Claude wastes tokens and makes it worse.

It edited the wrong file

Check your checkpoints and revert. Then be more specific: tell Claude exactly which file to touch and which to leave alone. Name the files explicitly.

It confidently made something up

Claude can hallucinate file names, function names, or steps that don't exist. Always verify output. If something looks wrong, it probably is — ask Claude to double-check its own work.

Nothing is working and you don't know why

Start a fresh session. Paste in a one-paragraph summary of what you were doing and what broke. Fresh context fixes more problems than you'd expect.

The golden rule: if Claude has failed the same task twice, don't try a third time the same way. Change your approach, simplify the ask, or break it into smaller steps.
