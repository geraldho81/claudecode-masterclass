The Product

What is Claude Code?

Claude Code is an AI tool you run from the terminal. It can read your project, edit files, run commands, and help complete real tasks.

You point it at a folder, tell it what to do, and it works through the files.

The Workspace

What is the Terminal?

The terminal is a text-based control panel for your computer. Instead of clicking buttons, you type instructions.

For non-technical people: it looks like a blank screen with a blinking cursor. After a few tries, the directness starts to make sense. You can see exactly what the computer is doing.

Developer Environment

Why Use the Terminal?

The terminal is where developers already build, test, and run things. Claude Code lives there too, so it can read and edit your actual files rather than describing what you should do yourself.

By living in the terminal, Claude doesn't just write code in a browser window for you to copy-paste. It writes the code directly into your actual files.

Quick Start

Setup:
Simple, Not Always Fast

Three commands. But on Windows especially, getting Node.js, npm, and the terminal playing nicely can take a while. Budget 20–30 minutes your first time.

$ cd any-project-folder

> Ready to work.

Mac users: usually smooth. Windows users: expect some friction. It's worth it once it's done.

Session Hygiene

Managing Your Context

/clear

Wipes the conversation history without closing Claude. Use it between unrelated tasks. It's the highest-ROI habit you can build.

/compact

Summarises the conversation to free up context space. Use mid-session when you want to keep going without starting over.

Fresh Session

After two failed corrections, start a new session. Give Claude a one-paragraph recap. A clean start beats a broken conversation every time.

Think of context like RAM. The more you load in, the slower and fuzzier things get. Keep sessions focused and short.

The Magic Command

Pair it with a CLAUDE.md brain file for best results.

Configuration

Personal Rules vs.
Project Rules

Global: ~/.claude.md

Your personal defaults. Think of it as a brain file. Keep it to ~1 page so Claude doesn't forget what matters.

Project: /project-folder/CLAUDE.md

Instructions for one specific project so Claude follows the right rules every session.

Power Feature

Working with Skills

A Skill is a saved instruction set for a task you run regularly. Instead of re-explaining the process each time, you point Claude at the Skill and it follows the same steps every session.

Run the same SEO audit or code review every week? Save it once. Claude reads the Skill file and runs it the same way each time.

Security

Keeping It Safe

Claude Code is not magic access to your whole computer. It works within your permissions and workspace rules, and sandboxing can restrict what it can change outside the current directory.

Don't worry that it's going to delete your operating system. It works on the folder you put it in.

Safety Net

Checkpoints:
Your Undo Button

Before big changes, Claude saves a checkpoint. If something goes wrong, press Esc twice or type /rewind to roll back instantly.

Restore code, conversation, or both. Think of it like a save point in a video game. No Git expertise required.

Control

You Are Still the Boss

Claude works at speed, but every change it makes is something you approved. Review before accepting. Redirect when it drifts.

It's an assistant, not an autopilot.

Reality Check

The Honest Truth

CLAUDE.md hints fade over time. Long sessions = fading memory. Keep your brain file to ~1 page.

Use /clear between unrelated tasks. It's the single highest-ROI habit you can build.

After two failed corrections, start fresh. A clean prompt beats arguing in circles.

Always verify the work. Claude is enthusiastic, fast, and occasionally wrong.

Tokens burn fast. Intensive work like coding can drain your token limit quickly, especially on Pro. Always plan your task before you start, not halfway through.

Troubleshooting

When Things Go Wrong

Claude is going in circles

Stop it. Type /clear and restate the goal more simply. Arguing with a looping Claude wastes tokens and makes it worse.

It edited the wrong file

Check your checkpoints and revert. Then be more specific: tell Claude exactly which file to touch and which to leave alone.

It confidently made something up

Claude can hallucinate file names or steps that don't exist. Always verify the output. If something looks wrong, it probably is.

Nothing is working and you don't know why

Start a fresh session. Paste in a one-paragraph summary of what you were doing and what broke. Fresh context fixes more than you'd expect.

The golden rule: if Claude has failed the same task twice, don't try a third time the same way. Change your approach, simplify the ask, or break it into smaller steps.
